# Registro De Commits

## Objetivo

Mantener un historial de cambios claro para evidenciar el avance del proyecto.

## Estrategia

Los commits se separan por entregables:

- Backend base.
- Pagina funcional.
- Login y control por roles.
- Documentacion academica.
- Evidencias de prueba.

## Verificacion

Comando para contar commits:

```powershell
git rev-list --count main
```
