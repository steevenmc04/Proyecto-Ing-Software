# Registro De Commits

## Objetivo

Mantener un historial de cambios claro para evidenciar el avance del proyecto.

## Estrategia

Los commits se separan por entregables:

- Backend base.
- Pagina funcional.
- Login y control por roles.
- Documentacion academica.
- Evidencias de prueba.

## Verificacion

El repositorio debe conservar como minimo 20 commits para evidenciar el avance
iterativo del desarrollo. El conteo actual se debe consultar directamente desde
Git para evitar que este documento quede desactualizado.

Comando para contar commits:

```powershell
git rev-list --count main
```
